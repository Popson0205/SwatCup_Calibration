Quick Start Guide

1. Setup

Upload your SWAT output files (.rch, file.cio) and observed flow CSVs. Set your calibration period dates.

2. Select Parameters

Use the searchable parameter picker to add parameters. The top 5 most important SWAT parameters are pre-loaded. Adjust min/max ranges as needed.

3. Calibration

Set number of simulations (500–1000 recommended) and run calibration. SUFI-2 uses Latin Hypercube Sampling to explore the parameter space.

4. Validation

Apply the best parameter set to an independent validation period to confirm model performance.

5. Results

View NSE, KGE, PBIAS metrics and interactive charts. Download the full results ZIP.

NSE Difference (Online vs Desktop)

SUFI-2 uses random Latin Hypercube Sampling — each run explores a different region of parameter space. Different runs will produce different NSE values. This is expected behaviour, not a bug. Run more simulations (1000+) for more consistent results.

File Formats

output.rch — SWAT reach output file. Run SWAT once to generate it before calibrating.

file.cio — SWAT master watershed file. Used to read IYR, NBYR, NYSKIP for date alignment.

observed_flow_rchN.csv — Observed streamflow. Two formats accepted:
Narrow: date, flow
Wide: date, rch13, rch15, rch16, rch17

par_inf.txt — Optional custom parameter ranges. If omitted, built-in 71-parameter defaults are used.

ℹ NSE Consistency Check

Why do Online and Desktop results differ?

SUFI-2 uses Latin Hypercube Sampling (LHS) — each run randomly samples the parameter space. Two independent runs will explore different parameter combinations and produce different NSE values.

This is expected and correct behaviour.

The NSE formula used:

NSE = 1 - Σ(Qobs - Qsim)² / Σ(Qobs - mean(Qobs))²

To get consistent results:

  • Use more simulations (1000+)
  • Run multiple iterations
  • Narrow parameter ranges after each iteration
